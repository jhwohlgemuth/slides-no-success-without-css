PostCSS

A tool for transforming CSS with JavaScript

  • Find and fix CSS syntax errors with postcss-safe-parser
  • Add vendor prefixes auto-magically with autoprefixer
  • Lint your stylesheets with stylint
  • Break up styles into separate files with postcss-import
  • Use tomorrow's CSS today with postcss-cssnext
  • Minimize your stylesheets with cssnano

### [PostCSS](http://postcss.org/) is the cure for many of today's CSS headaches

Stop Writing Vendor Prefixes

✗Problem Developers add prefixes we don't need and forget ones we do...
✔Solution Add vendor prefixes dynamically for your target browser(s) using postcss-cssnext

Break up your files

✗Problem style.css that scrolls for days...merge conflicts that make you question life choices
✔Solution Design and implement a folder structure. Import CSS files "like JS modules" using postcss-import

Start nesting classes!

✗Problem CSS just kind of sucks can be redundant
✔Solution "DRY" up your CSS with nested classes and variables using postcss-cssnext

Make your CSS more robust

✗Problem CSS just kind of sucks tends to hide issues
✔Solution Parse CSS with postcss-safe-parser and lint CSS with stylelint

ES6 Template Literals

Template Literals

String interpolation


    // ES5
    var fullName = last + ', ' + first + ' ' + middleInitial;
            

    // ES6
    const fullName = `${last}, ${first} ${middleInitial}`;

Template Literals

Multi-line strings


    // ES5
    var message = '' +
        'Dear Santa,\n' +
        '    Cookies are insecure and have been removed.\n' +
        'Sincerely,\n' +
        'Little Timmy, CISSP'
            

    // ES6
    var message = `
    Dear Santa,
        Cookies are insecure and have been removed.
    Sincerely,
    Little Timmy, CISSP`

        const truth = 'JavaScript';
        language = (language !== truth) ? truth : language;
        return text[0] + language;
    };
